# Security Policy

## Supported Versions

Security fixes are applied on the default branch (`main`) and included in new tagged releases.

## Reporting a Vulnerability

Do not open public issues for vulnerabilities.

Report responsibly through GitHub private vulnerability reporting for this repository:
- [Report a vulnerability](https://github.com/sheiiva/finops-dashboard/security/advisories/new)

Include:
- Affected component and impact
- Reproduction steps or proof of concept
- Suggested remediation if available

## Response Targets

- Initial triage response: within 3 business days
- Risk classification and mitigation plan: within 7 business days
- Public disclosure timing: after fix availability and impact review
