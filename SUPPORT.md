# Support

## How to Get Help

- Use GitHub Issues for bugs, feature requests, and implementation tasks.
- Use issue templates to provide required context and acceptance criteria.
- Link related milestones and pull requests for traceability.

## Response Model

- Acknowledgement target: within 3 business days.
- Critical defects and security-impacting concerns are prioritized first.
- Security vulnerabilities must be reported privately via `SECURITY.md`.

## What to Include

- Problem summary and expected behavior
- Reproduction steps and logs/screenshots when relevant
- Scope (cloud, environment, module)
- Business/operational impact
